    1Temporal.Date Objects

    A Temporal.Date object is an immutable Object that contains Number values corresponding to a particular year, month, and day.

    1.1The Temporal.Date Constructor

    The Temporal.Date constructor is the %Temporal.Date% intrinsic object. When called as a constructor, it creates and initializes a new Temporal.Date object.

    The Temporal.Date constructor is designed to be subclassable. It may be used as the value of an extends clause of a class definition. Subclass constructors that intend to inherit the specified Date behaviour must include a super call to the %Temporal.Date% constructor to create and initialize subclass instances with the necessary internal slots.

    1.1.1Temporal.Date ( year, month, day, disambiguation )

    When the Temporal.Date function is called, the following steps are taken:

    1. If NewTarget is undefined, then
      1. Throw a TypeError exception.
    2. Let y be ? ToInteger(year).
    3. Let m be ? ToInteger(month).
    4. Let d be ? ToInteger(day).
    5. Let disambiguation be ? ToString(disambiguation).
    6. Let result be ? RegulateDate(y, m, d, disambiguation).
    7. Return ? CreateDate(result.[[Year]], result.[[Month]], result.[[Day]], NewTarget).

    1.2Properties of the Temporal.Date Constructor

    The value of the [[Prototype]] internal slot of the Temporal.Date constructor is the intrinsic object %FunctionPrototype%.

    The Temporal.Date constructor has the following properties:

    1.2.1Temporal.Date.prototype

    The initial value of Temporal.Date.prototype is %Temporal.Date.prototype%.

    This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

    1.2.2Temporal.Date.fromString ( isostring )

    The fromString method takes one argument isostring. The following steps are taken:

    1. Let validSyntax be true if isostring conforms to the format YYYY-MM-DD, and false otherwise, where:
      • YYYY is either four decimal digits or an expanded year of "+" or "-" followed by six decimal digits;
      • - is the code unit 0x002D (HYPHEN-MINUS);
      • MM is two decimal digits;
      • - is the code unit 0x002D (HYPHEN-MINUS);
      • DD is two decimal digits.
    2. If validSyntax is false, then
      1. Throw a RangeError exception.
    3. Let year, month, and day be the respective parts of isostring, interpreted as decimal numbers.
    4. Let result be ? RegulateDate(year, month, day, "reject").
    5. Return ? CreateDate(result.[[Year]], result.[[Month]], result.[[Day]]).

    1.2.4Temporal.Date.compare ( one, two )

    The compare method takes two arguments, one and two. The following steps are taken:

    1. Let one be ? ToDate(one).
    2. Let two be ? ToDate(two).
    3. If one.[[Year]] > two.[[Year]], return 1.
    4. If one.[[Year]] < two.[[Year]], return -1.
    5. If one.[[Month]] > two.[[Month]], return 1.
    6. If one.[[Month]] < two.[[Month]], return -1.
    7. If one.[[Day]] > two.[[Day]], return 1.
    8. If one.[[Day]] < two.[[Day]], return -1.
    9. Return +0.

    1.4Properties of Temporal.Date Instances

    Temporal.Date instances are ordinary objects that inherit properties from the %Temporal.Date.prototype%.

    Temporal.Date instances have the following internal slots:

    • [[Year]], representing the year.
    • [[Month]], representing the month within the year. It lies between 1 and 12, inclusive.
    • [[Day]], representing the day within the month. It lies between 1 and DaysInMonth([[Year]], [[Month]]), inclusive.

    The value of each of these internal slots is a Number which is an integer.

    1.5.2.3ValidateDate ( y, m, d )

    1. Assert: y, m, and d are integer Number values.
    2. If m < 1 or m > 12, then
      1. Return false.
    3. Let maxDay be DaysInMonth(y, m).
    4. If d < 1 or d > maxDay, then
      1. Return false.
    5. Return true.

    1.5.2.4PadYear ( y )

    1. Assert: y is an integer Number value.
    2. If y > 999 and y ≤ 9999, then

    An Instant is an immutable object referencing a point in time with nanoseconds precision.

    4.1new Instant(nanoseconds)

    When called as a constructor creates a new Instant object. nanoseconds is a BigInt of the nanoseconds since epoch.

    4.2Instant.prototype.epochSeconds

    instant.epochSeconds is a getter that returns the seconds Number integer since epoch of the point in time represented by the instant object such that instant.epochSeconds === Number(instant.valueOf() / BigInt(1e9) is true.

    4.3Instant.prototype.epochMilliseconds

    instant.epochMilliseconds is a getter that returns the milliseconds Number integer since epoch of the point in time represented by the instant object such that instant.epochMilliseconds === Number(instant.valueOf() / BigInt(1e6)) is true.

    4.4Instant.prototype.epochMicroseconds

    instant.epochMicroseconds is a getter that returns the microseconds BigInt since epoch of the point in time represented by the instant object such that instant.epochMicroseconds === Number(instant.valueOf() / BigInt(1e3)) is true.

    4.5Instant.prototype.epochNanoseconds

    instant.epochNanoseconds is a getter that returns the nanoseconds BigInt since epoch of the point in time represented by the instant object such that instant.epochNanoseconds === instant.valueOf() is true.

    4.6Instant.prototype.withZone(timezone)

    When called with an Instant object as a this context it will return a ZonedDateTime object that specifies an instant in time in the given time zone.

    timezone is a string specifying a time zone. Legal values are IANA time zone names.

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.7Instant.prototype.withOffset(offset)

    When called with an Instant object as a this context it will return a OffsetDateTime object that specifies an instant in time in the given time zone.

    offset is a string specifying an offset as specified in OffsetDateTime

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.8Instant.prototype.toString()

    When called with an Instant object as a this context it will return an ISO 8601 String value representing the value of the Instant object.

    The format of the String will always be an IsoDateTime of the following form:

    Sign:- Year:SignoptDigitDigitDigitDigit Month:01

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.9Instant.fromString(string)

    This function will create an Instant object from a String. The string has to strictly conform to the format produced by . Any string not strictly conforming to that format will throw an Error.

    4.10Instant.fromEpochSeconds(seconds)

    This function will create an Instant object that represents the instant in time seconds since epoch such that BigInt(instant.epochSeconds) * BigInt(1e9) === instant.epochNanoseconds and instant.epochSeconds === _seconds_ are both true.

    4.11Instant.fromEpochMilliseconds(milliseconds)

    This function will create an Instant object that represents the instant in time milliseconds since epoch such that BigInt(instant.epochMilliseconds) * BigInt(1e6) === instant.epochNanoseconds and instant.epochMilliseconds === _milliseconds_ are both true.

    4.12Instant.fromEpochMicroseconds(microseconds)

    This function will create an Instant object that represents the instant in time milliseconds since epoch such that instant.epochMicroseconds * BigInt(1e3) === instant.epochNanoseconds and instant.epochMicroseconds === _microseconds_ are both true.

    4.13Instant.fromEpochNanoseconds(nanoseconds)

    This function will create an Instant object that represents the instant in time nanoseconds since epoch such that instant.epochNanoseconds === instant.epochNanoseconds and instant.epochMicroseconds === _microseconds_ are both true.

    2. Else,
      1. Let pattern be dateTimeFormat.[[Pattern]].
      2. If date is undefined, then
        1. Let x be Call(%Date_now%, undefined).
      3. Else,
        1. Let x be ? ToNumber(date).

        1. Perform ! CreateDataPropertyOrThrow(options, p, v).
    6. Return options.

    For web compatibility reasons, if the property hourCycle is set, the hour12 property should be set to true when hourCycle is "h11" or "h12", or to false when hourCycle is "h23" or "h24".

    Note 1
    In this version of the ECMAScript 2020 Internationalization API, the timeZone property will be the name of the default time zone if no timeZone property was provided in the options object provided to the Intl.DateTimeFormat constructor. The first edition left the timeZone property undefined in this case.
    Note 2
    For compatibility with versions prior to the fifth edition, the "hour12" property is set in addition to the "hourCycle" property.

    6.3Properties of Intl.DateTimeFormat Instances

    Intl.DateTimeFormat instances inherit properties from %DateTimeFormatPrototype%.

    Intl.DateTimeFormat instances have an [[InitializedDateTimeFormat]] internal slot.

    Intl.DateTimeFormat instances also have several internal slots that are computed by the constructor:

    • [[Locale]] is a String value with the language tag of the locale whose localization is used for formatting.
    • [[Calendar]] is a String value with the "type" given in Unicode Technical Standard 35 for the calendar used for formatting.
    • [[NumberingSystem]] is a String value with the "type" given in Unicode Technical Standard 35 for the numbering system used for formatting.
    • [[TimeZone]] is a String value with the IANA time zone name of the time zone used for formatting.
    • [[Weekday]], [[Era]], [[Year]], [[Month]], [[Day]], [[Hour]], [[Minute]], [[Second]], [[TimeZoneName]] are each either undefined, indicating that the component is not used for formatting, or one of the String values given in Table 2, indicating how the component should be presented in the formatted output.
    • [[HourCycle]] is a String value indicating whether the 12-hour format ("h11", "h12") or the 24-hour format ("h23", "h24") should be used. "h11" and "h23" start with hour 0 and go up to 11 and 23 respectively. "h12" and "h24" start with hour 1 and go up to 12 and 24. [[HourCycle]] is only used when [[Hour]] is not undefined.
    • [[Pattern]], [[CivilDatePattern]], … are is a String valuerecords containing at least a [[pattern]] field as described in Internal slots.

    Finally, Intl.DateTimeFormat instances have a [[BoundFormat]] internal slot that caches the function returned by the format accessor (get Intl.DateTimeFormat.prototype.format).

    7Abstract operations

    7.1IsLeapYear ( year )

    1. Assert: year is an integer.
    2. If year modulo 4 ≠ 0, return false.
    3. If year modulo 400 = 0, return true.
    4. If year modulo 100 = 0, return false.
    5. Return true.

    7.2DaysInMonth ( year, month )

    1. Assert: month is an integer, month ≥ 1, and month ≤ 12.
    2. If month is 1, 3, 5, 7, 8, 10, or 12, return 31.
    3. If month is 4, 6, 9, or 11, return 30.
    4. If ! IsLeapYear(year) is true, return 29.
    5. Return 28.

    7.3ToDayOfWeek ( year, month, day )

    1. Let date be the date given by year, month, and day.
    2. Return date's day of the week according to ISO-8601.
    Note
    Monday is 1 and Sunday is 7.

    7.4ToDayOfYear ( year, month, day )

    1. Let date be the date given by year, month, and day.
    2. Return date's ordinal date in the year according to ISO-8601.

    7.5ToWeekOfYear ( year, month, day )

    1. Let date be the date given by year, month, and day.
    2. Return date's week number according to ISO-8601.
    Note
    Beware that dates at the begining of a year may be part of a week from the preceding year, and dates at the end of a year may be part of a week at the beginning of the next year, as the first week of any year is defined as the week that contains the first Thursday of the year.
