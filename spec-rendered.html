    1CivilDate Objects

    A CivilDate object is an immutable Object that contains Number values corresponding to a particular year, month, and day.

    The following functions are abstract operations that operate on time values (defined SOMEWHERE). Note that, in every case, if any argument to one of these functions is NaN, the result will be NaN.

    1.1Year Number

    The Year Number is an integer Number corresponding to a particular year. It is called the year value. It must have a Number value and may not be NaN.

    1.2Month Number

    The Month Number is a positive integer Number corresponding to a particular month. It is called the month value and is identified by an integer in the range of 1 to 12, inclusive. It must have a Number value and may not be NaN.

    1.3Day Number

    The Day Number is a positive integer Number corresponding to a particular day of the month. It is called the day value and is identified by an integer the value of which is dependent on the month and year values. It must have a Number value and may not be NaN. The range of possible day values can be calculated as follows:

    1. For any year value with a month values of 1, 3, 5, 7, 8, 10, or 12, the day value is in the range of 1 to 31, inclusive.
    2. For any year value with a month value of 4, 6, 9, or 11, the day value is in the range of 1 to 30, inclusive.
    3. For the month value 2, the range of possible values for the day value depends on whether the year value corresponds to a leap year.
      1. If the year value modulo 4 != 0, the range is 1 to 28, inclusive.
      2. If the year value modulo 400 == 0, the range is 1 to 29, inclusive.
      3. If the year value modulo 100 == and year value % 400 != 0, the range is 1 to 28, inclusive.
      4. If the year value modulo 4 == 0 and year value % 100 != 0, the range is 1 to 29, inclusive.

    1.4Plus(Object)

    The abstract operation Plus returns a new CivilDate object that may have 0 to 2 values copied from the CivilDate object on which the operation was performed. The new CivilDate object will have one or more updated values that depend on terminal values of an Object passed in as the argument. The argument Object consists of a range of 1 to 3 (inclusive) key-value pairs. Each pair consists of a nonterminal symbol and a corresponding terminal Number. An argument Object may have one and only one of each of the nonterminal symbols "Year", "Month", and "Day".

    1.5With(?_year, ?_month, ?_day)

    The abstract operation With takes one Object argument and returns a new CivilDate object. The Object argument must have a range of 1 to 3 inclusive key/value pairs with the keys "year", "month", and/or "day". The corresponding value for each key must be a Number.

    The new CivilDate object copies the YearValue, MonthValue, and/or DayValue properties from the original CivilDate object and replaces the property value(s) with the corresponding argument value.

    1.6WithTime(time)

    The abstract operation WithTime returns a CivilDateTime object that copies and may update the year, month, and day values of the CivilDate object on which the operation was performed.

    1.7CivilDate Constructor

    The CivilDate constructor is the %CivilDate% intrinsic object. When called as a constructor, it creates and initializes a new CivilDate object.

    The CivilDate constructor is a single function.

    The CivilDate constructor is subclassable

    The length property of the CivilDate constructor function is 3.

    1.8CivilDate (year, month, day)

    This description only applies if the CivilDate constructor is called with at least 3 arguments.

    When the CivilDate function is called, the following steps are taken:

    1. Let numberOfArgs be the number of arguments passed to this function call.
    2. Assert: numberOfArgs == 3.
    3. If NewTarget is not undefined, then
      1. Let y be ? ToNumber(year)
      2. Let m be ? ToNumber(month)
      3. Let d be ? ToNumber(Day)

    1.11CivilDate.plus (data)

    The abstract operation plus takes as an argument an Object with a range of 1 to 3 key/value pairs, inclusive.

    The keys may not be duplicated and may be any combination of "year", "month", "day".

    The values must be of the Number type and may be positive or negative integers.

    If the sum of the current value and the argument value exceeds the range of allowable values for the given unit, increment the next unit.

    The CivilDate operation uses Gregorian pro-leptic calendar computation.

    When the plus function is called, the following steps are taken:

    1. Let numberOfArgs be the number of arguments passed to this function call.
    2. Assert: Type(data) is Object.
    3. Assert: numberOfArgs == 1.
    4. If NewTarget is not undefined, then
      1. If data.year is supplied, let y be ? ToNumber(year) + thisYearValue. If y is NaN, throw a TypeError exception; else if data.year is not supplied, let y be thisYearValue.
      2. If data.month is supplied, let m be ? ToNumber(month) + thisMonthValue. If m is NaN, throw a TypeError exception; else if data.month is not supplied, let m be thisMonthValue.
      3. If data.day is supplied, let d be ? ToNumber(day) + thisDayValue. If d is NaN, throw a TypeError exception; else if data.day is not supplied, let d be thisDayValue.
      4. Regulate the final y, m, and d values using Gregorian pro-leptic calendar calculations.
      5. Let O be ? OrdinaryCreateFromConstructor(New Target, "%CivilDatePrototype%", y, m, d)
      6. Set O.[[YearValue]] to y.
      7. Set O.[[MonthValue]] to m.
      8. Set O.[[DayValue]] to d.
      9. Return O.

    The following functions are abstract operations that operate on time values (defined SOMEWHERE). Note that, in every case, if any argument to one of these functions is NaN, the result will be NaN.

    2.1Hour Number

    The Hour Number is a positive integer Number corresponding to a particular hour. It is called the HourValue and is identified by an integer in the range of 0 to 23, inclusive. It must have a Number value and may not be NaN.

    2.2Minute Number

    The Minute Number is a positive integer Number corresponding to a particular minute. It is called the MinuteValue and is identified by an integer in the range of 0 to 59, inclusive. It must have a Number value and may not be NaN.

    2.3Second Number

    The Second Number is a positive integer Number corresponding to a particular second. It is called the SecondValue and is identified by an integer in the range of 0 to 59, inclusive. It must have a Number value and may not be NaN.

    2.4Millisecond Number

    The Millisecond Number is a positive integer Number corresponding to a particular millisecond. It is called the MillisecondValue and is identified by an integer in the range of 0 to 999, inclusive. It must have a Number value and may not be NaN.

    2.5Nanosecond Number

    The Nanosecond Number is a positive integer Number corresponding to a particular nanosecond. It is called the NanosecondValue value and is identified by an integer in the range of 0 to 999999, inclusive. It must have a Number value and may not be NaN.

    2.6Plus(Object)

    The abstract operation Plus returns a new CivilTime object that may have 0 to 4 values copied from the CivilTime object on which the operation was performed. The new CivilTime object will have one or more updated values that depend on terminal values of an Object passed in as the argument. The argument Object consists of a range of 1 to 5 (inclusive) key/value pairs. Each pair consists of a nonterminal symbol and a corresponding terminal Number.     2.7With(?_hour, ?_minute, ?_second, ?_millisecond, ?_nanosecond)

    The abstract operation With takes one Object argument and returns a new CivilTime object.

    The Object argument must have a range of 1 to 5 inclusive key/value pairs with the keys "hour", "minute", "second", "millisecond", and/or "nanosecond".

    The corresponding value for each key must be a Number.

    The new CivilTime object copies the HourValue, MinuteValue, SecondValue, MillisecondValue, and/or NanosecondValue properties from the original CivilTime object and replaces the property value(s) with the corresponding argument value.

    2.8WithDate(date)

    The abstract operation WithTime returns a CivilDateTime object with the date values passed as the argument and the hour, minute, second, millisecond, and nanosecond values of the CivilTime object on which the operation was performed.

    2.10CivilTime (hour, minute[[[, second], millisecond], nanosecond])

    This description only applies if the CivilTime constructor is called with at least two arguments.

    When the CivilTime function is called, the following steps are taken:

    1. Let numberOfArgs be the number of arguments passed to this function call.
    2. Assert: numberOfArgs >= 2.
    3. If hour is not finite or minute is not finite or second is not finite or millisecond is not finite or nanosecond is not finite, return NaN.
    4. If NewTarget is not undefined, then
      1. Let h be ? ToNumber(hour).
      2. Let min be ? ToNumber(minute).
      3. If "second" is supplied, let s be ? ToNumber(second); else let s be 0.
      4. If "millisecond" is supplied, let ms be ? ToNumber(millisecond); else let ms be 0.
      5. If "nanosecond" is supplied, let ns be ? ToNumber(nanosecond); else let ns be 0.
      6. Let O be ? OrdinaryCreateFromConstructor(NewTarget, "%CivilTimePrototype%", <<[[HourValue, MinuteValue, SecondValue, MillisecondValue, NanosecondValue]]>>).
      7. Set O.[[HourValue]] to h.
      8. Set O.[[MinuteValue]] to min.
      9. Set O.[[SecondValue]] to s.
      10. Set O.[[MillisecondValue]] to ms.
      11. Set O.[[NanosecondValue]] to ns.
      12. Return O

    4Instant Objects

    An Instant is an immutable object referencing a point in time with nanoseconds precision.

    4.1new Instant(nanoseconds)

    When called as a constructor creates a new Instant object. nanoseconds is a BigInt of the nanoseconds since epoch.

    4.2Instant.prototype.valueOf()

    When called with an Instant object as the this context the result will be a BigInt value with the nanoseconds since epoch the instant represents.

    Calling this function with anything other than an Instant object as the this context is illegal.

    4.3Instant.prototype.milliseconds

    instant.milliseconds is a getter that returns the milliseconds Number integer since epoch of the point in time represented by the instant object such that instant.milliseconds === Math.floor(instant.valueOf() / 1e6) is true.

    4.4Instant.prototype.nanoseconds

    instant.nanoseconds is a getter that returns the nanoseconds Number such that it represents only the nanoseconds that are not represented in Instant.prototype.milliseconds such that instant.nanoseconds === instant.valueOf() % 1e6 is true.

    4.5Instant.prototype.plus({ ? years, ? months, ? days, ? hours, ? minutes, ? seconds, ? milliseconds, ? nanoseconds })

    When called with an Instant object as the this context the result will be a new Instant object. The value of the new Instant object is the value of the original instant with the specified date parts added. Units will be added in order of size, descending.

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.6Instant.prototype.withZone(timezone)

    When called with an Instant object as a this context it will return a ZonedInstant object that specifies an instant in time in the given time zone.

    timezone is a string specifying a time zone. Legal values are IANA geographic time zone names as well as the special values "SYSTEM" and "UTC".

    • "UTC" indicates that the time zone should be Coordinated Universal Time.
    • "SYSTEM" indicates that the time zone should be local time as provided by the system.

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.7Instant.prototype.toString()

    When called with an Instant object as a this context it will return an ISO 8601 String value representing the value of the Instant object.

    The format of the String will always be an IsoDateTime of the following form:

    Sign:- Year:SignoptDigitDigitDigitDigit Month:01

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.8Instant.prototype.format([locale], [options])

    When called with an Instant object as a this context it will return an String value representing the value of the Instant object as formated by an Intl.DateFormatter objects format method when invoked with the locale and options arguments passed through.

    The date argument to the Intl.DateDormatter is equivalent to new Date(instant.milliseconds).

    Calling this function with a this context that is not an instance of Instant is illegal.

    4.9Instant.fromMilliseconds(miliseconds)

    This function will create an Instant object that represents the instant in time milliseconds since epoch. Nanoseconds will be 0.

    milliseconds is the Number of milliseconds since epoch.

    4.10Instant.fromString(string)

    This function will create an Instant object from a String. The string has to strictly conform to the format produced by . Any string not strictly conforming to that format will throw an Error.

    5ZonedInstant Objects

    A ZonedInstant object is an immutable object referencing a point in time. The referenced point in time is specific to a given time zone and is measured with nanoseconds precision.

    5.1Millisecond Number

    The Millisecond Number is an integer corresponding to a particular millisecond. It is called the millisecond value. It must have a Number value and may not be NaN.

    5.2Nanosecond Number

    The Nanosecond Number is an integer corresponding to a particular nanosecond. It is called the nanosecond value and is identified by an integer in the range of 0 to 999999, inclusive. It must have a Number value and may not be NaN.

    5.3TimeZone String

    The TimeZone String is an string corresponding to a particular IANA time zone. It is called the timeZone value. It must have a String value and may not be null.

    5.4ZonedInstant Prototype

    The initial value of ZonedInstant.prototype is the intrinsic object %ZonedInstantPrototype%. This property has the attributes {[[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false}.

    5.5ZonedInstant.prototype.valueOf()

    When called with a ZonedInstant object as the this context the result will be a BigInt value with the nanoseconds since epoch the ZonedInstant represents.

    Calling this function with anything other than a ZonedInstant object as the this context is illegal.

    5.6ZonedInstant.prototype.milliseconds

    ZonedInstant.milliseconds is a getter that returns the milliseconds Number integer since epoch of the point in time represented by the ZonedInstant object such that ZonedInstant.milliseconds === Math.floor(ZonedInstant.valueOf() / 1e6) is true.

    5.7ZonedInstant.prototype.nanoseconds

    ZonedInstant.nanoseconds is a getter that returns the nanoseconds Number such that it represents only the nanoseconds that are not represented in ZonedInstant.prototype.milliseconds such that ZonedInstant.nanoseconds === ZonedInstant.valueOf() % 1e6 is true.

    5.8ZonedInstant.prototype.timeZone

    ZonedInstant.timeZone is a getter that returns the object's associated IANA time zone value as a String.

    5.9ZonedInstant.prototype.with({ ? years, ? months, ? days, ? hours, ? minutes, ? seconds, ? milliseconds, ? nanoseconds })

    >When called with a ZonedInstant object as the this context, the result will be a new ZonedInstant object.

    The value of the new ZonedInstant object is the value of the original ZonedInstant's milliseconds with the specified date parts, excepting any nanoseconds argument, converted to milliseconds and added to the original MillisecondValue.

    Units will be added in order of size, descending. Calling this function with a this context that is not an instance of ZonedInstant is illegal.

    5.10ZonedInstant.prototype.plus({ ? years, ? months, ? days, ? hours, ? minutes, ? seconds, ? milliseconds, ? nanoseconds })
