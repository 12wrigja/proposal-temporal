Instant Objects

An `Instant` is an immutable object referencing a point in time with nanoseconds precision.

new Instant(*nanoseconds*)

When called as a constructor creates a new Instant object. *nanoseconds* is a BigInt of the nanoseconds since epoch.

Instant.prototype.epochSeconds

`instant.epochSeconds` is a getter that returns the seconds Number integer since epoch of the point in time represented by the instant object such that `instant.epochSeconds === Number(instant.valueOf() / BigInt(1e9)` is *true*.

Instant.prototype.epochMilliseconds

`instant.epochMilliseconds` is a getter that returns the milliseconds Number integer since epoch of the point in time represented by the instant object such that `instant.epochMilliseconds === Number(instant.valueOf() / BigInt(1e6))` is *true*.

Instant.prototype.epochMicroseconds

`instant.epochMicroseconds` is a getter that returns the microseconds BigInt since epoch of the point in time represented by the instant object such that `instant.epochMicroseconds === Number(instant.valueOf() / BigInt(1e3))` is *true*.

Instant.prototype.epochNanoseconds

`instant.epochNanoseconds` is a getter that returns the nanoseconds BigInt since epoch of the point in time represented by the instant object such that `instant.epochNanoseconds === instant.valueOf()` is *true*.

Instant.prototype.withZone(_timezone_)

When called with an Instant object as a _this_ context it will return a ZonedDateTime object that specifies an instant in time in the given time zone.

_timezone_ is a string specifying a time zone. Legal values are IANA geographic time zone names as well as the special values *"SYSTEM"* and *"UTC"*.

  • *"UTC"* indicates that the time zone should be Coordinated Universal Time.
  • *"SYSTEM"* indicates that the time zone should be local time as provided by the system.

Calling this function with a _this_ context that is not an instance of Instant is illegal.

Instant.prototype.valueOf()

When called with an Instant object as the _this_ context the result will be a BigInt value with the nanoseconds since epoch the instant represents.

Calling this function with anything other than an Instant object as the _this_ context is illegal.

Instant.prototype.toString()

When called with an Instant object as a _this_ context it will return an ISO 8601 String value representing the value of the Instant object.

The format of the String will always be an IsoDateTime of the following form:

Instant.fromString(_string_)

This function will create an Instant object from a String. The _string_ has to strictly conform to the format produced by . Any _string_ not strictly conforming to that format will throw an Error.

Instant.fromSeconds(_seconds_)

This function will create an Instant object that represents the instant in time _seconds_ since epoch such that `BigInt(instant.epochSeconds) * BigInt(1e9) === instant.epochNanoseconds` and `instant.epochSeconds === _seconds_` are both `true`.

Instant.fromMilliseconds(_miliseconds_)

This function will create an Instant object that represents the instant in time _milliseconds_ since epoch such that `BigInt(instant.epochMilliseconds) * BigInt(1e6) === instant.epochNanoseconds` and `instant.epochMilliseconds === _milliseconds_` are both `true`.

Instant.fromMicroseconds(_microseconds_)

This function will create an Instant object that represents the instant in time _milliseconds_ since epoch such that `instant.epochMicroseconds * BigInt(1e3) === instant.epochNanoseconds` and `instant.epochMicroseconds === _microseconds_` are both `true`.

Instant.fromNanoSeconds(_nanoseconds_)

This function will create an Instant object that represents the instant in time _nanoseconds_ since epoch such that `instant.epochNanoseconds === instant.epochNanoseconds` and `instant.epochMicroseconds === _microseconds_` are both `true`.
